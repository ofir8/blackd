# הוראות העלאה לשרת DirectAdmin

מדריך זה מסביר כיצד להעלות את האתר לשרת DirectAdmin שלך.

## שלב 1: בניית האתר

האתר כבר נבנה! כל הקבצים נמצאים בתיקייה `dist/`.

## שלב 2: התחברות ל-DirectAdmin

1. היכנס ל-DirectAdmin שלך דרך הדפדפן:
   ```
   https://your-domain.com:2222
   ```

2. התחבר עם פרטי ההתחברות שלך

## שלב 3: העלאת הקבצים

### אפשרות א: File Manager ב-DirectAdmin (הכי קל)

1. ב-DirectAdmin, לחץ על **File Manager**
2. נווט לתיקיית ה-`public_html` (או `domains/your-domain.com/public_html`)
3. **מחק** את כל הקבצים הקיימים (אם יש)
4. לחץ על **Upload** או **Upload Files**
5. בחר את **כל הקבצים** מתיקיית `dist/` שלך:
   - בחר את כל הקבצים והתיקיות מ-`dist/`
   - העלה אותם ל-`public_html`

### אפשרות ב: FTP (FileZilla, WinSCP)

1. התקן תוכנת FTP כמו FileZilla או WinSCP
2. התחבר ל-FTP עם פרטי ההתחברות מ-DirectAdmin:
   - **Host**: כתובת ה-FTP (לרוב: `ftp.your-domain.com` או כתובת ה-IP)
   - **Username**: שם המשתמש שלך
   - **Password**: הסיסמה שלך
   - **Port**: 21 (או הפורט שצוין)
3. נווט לתיקיית `public_html` בשרת
4. העלה את כל התוכן מ-`dist/` ל-`public_html`

### אפשרות ג: SCP (ממחשב Linux/Mac)

```bash
scp -r dist/* username@your-server-ip:/home/username/domains/your-domain.com/public_html/
```

## שלב 4: העלאת קובץ .htaccess

**חשוב מאוד!** קובץ ה-`.htaccess` חיוני לפעולה תקינה של האתר.

1. בתיקיית `dist/`, תראה קובץ בשם `.htaccess` (או `public/.htaccess`)
2. העלה אותו גם ל-`public_html`
3. ודא שהקובץ נמצא בתיקיית השורש של הדומיין

**הערה**: אם אתה משתמש ב-File Manager, ודא שהעלית גם קבצים נסתרים (hidden files)!

## שלב 5: בדיקת הרשאות

1. ב-File Manager, ודא שהרשאות הקבצים נכונות:
   - תיקיות: `755` או `750`
   - קבצים: `644` או `640`
2. אם צריך לשנות הרשאות:
   - לחץ ימין על הקובץ/תיקייה → **Change Permissions**

## שלב 6: בדיקה

1. פתח את הדפדפן ונווט לדומיין שלך
2. האתר אמור להיפתח בצורה תקינה
3. נסה לנווט בין הדפים השונים (סוויטה, צימר, וכו')

## בעיות נפוצות

### דף 404 על דפים פנימיים
- **פתרון**: ודא שקובץ `.htaccess` נמצא בתיקיית השורש
- ודא ש-mod_rewrite פעיל בשרת (רוב השרתים כבר פעיל)

### קבצי CSS/JS לא נטענים
- **פתרון**: בדוק שהקבצים בתיקייה `assets/` הועלו
- בדוק את הרשאות הקבצים

### האתר לא נטען בכלל
- **פתרון**: ודא שהקבצים נמצאים ב-`public_html` ולא בתת-תיקייה
- בדוק את ה-HTTP Error Logs ב-DirectAdmin

### שגיאת 403 Forbidden
- **פתרון**: בדוק את הרשאות התיקיות והקבצים
- ודא שיש קובץ `index.html` בתיקיית השורש

## עדכון האתר בעתיד

כאשר אתה רוצה לעדכן את האתר:

1. בצע את השינויים בקוד המקומי
2. הרץ:
   ```bash
   npm run build
   ```
3. העלה מחדש את כל התוכן מתיקיית `dist/` ל-`public_html`

## תמיכה

אם יש בעיות, בדוק:
- ה-HTTP Error Logs ב-DirectAdmin
- ה-HTTP Access Logs ב-DirectAdmin
- את קונסולת הדפדפן (F12) לבדיקת שגיאות JavaScript
